用户输入 `/check` 时触发。每日 15:27 自动触发（周一至周五）。

## 执行步骤

```bash
cd /Users/zhaojunjie/pyProject/cycleLaboratory && python3 scripts/fetch.py --all 2>&1
```
缓存策略：收盘后 15:02 定时任务会强制拉取一次。手动 `/check` 走缓存——交易时段自动过期、收盘后自动复用，不会重复拉。

### 2. 跑全量分析
```bash
cd /Users/zhaojunjie/pyProject/cycleLaboratory && python3 scripts/analyze.py --all 2>&1
```

注意：脚本 stdout 最后一行是 JSON 数组，其余为终端进度输出。用 `tail -1` 提取 JSON。

### 3. 输出格式
极简表格，异常标记必须打在行内，行动列必须给出具体操作（不能只写"观望"）：

```
名称           代码      现价     PB分位     RSI      均线      周期阶段          持仓   行动
════════════════════════════════════════════════════════════════════════════════════════════
酒ETF         512690   0.453   🔵0.4%     28.0     空头排列   底部（早期）       -     建观察仓
牧原股份       002714  37.760   17.2%     🔵15.6    空头排列   底部（早期）       -     等信号
紫金矿业       601899  30.710   🔴91.1%    30.9     空头排列   🔴顶部             -     不参与
沪深300ETF     510300   4.932   🔴82.9%    53.9     交叉      🔴顶部             -     不追高
```

表格规则：
- **异常标记打在具体数值/字段上**，不在行首名称上打，每行可出现多个标记
- **行动列必须给出具体操作**：建观察仓 / 等信号 / 不参与 / 可加仓 / 减仓 / 清仓 等，禁止笼统写"观望""持有"
- 每个标的下方附一句话操作建议 + 反方观点

持仓列（来自 ledger.yaml 假设交易记录）：
- 「-」— 无 AI 推荐持仓
- 「⚪ 观察仓 0.4万(+2.1%)」— 已推荐建观察仓，投入金额，当前浮盈/浮亏
- 「● 1/3仓 3.3万(-5.2%)」— 已推荐第一批建仓
- 「● 2/3仓 6.7万(+1.3%)」— 已推荐第二批加仓
- 「⚫ 满仓 10万(-8.3%)」— 已推满仓
- 「已清仓」— 已推荐全部卖出

### 4. 异常标记（打在具体数据字段上）

- RSI < 20 → RSI 数值前加 🔵，如 `🔵15.6`
- RSI > 75 → RSI 数值前加 🔴，如 `🔴78.2`
- PB分位 < 5% → PB分位前加 🔵，如 `🔵0.4%`
- PB分位 > 80% → PB分位前加 🔴，如 `🔴91.1%`
- 底部（确认）→ 周期阶段后加 ✅，如 `底部（确认）✅`
- 接飞刀 → 周期阶段前加 ⚠️，如 `⚠️接飞刀`
- 顶部 → 周期阶段前加 🔴，如 `🔴顶部`
- 假设持仓浮亏 > 10% → 持仓列后加 💀
