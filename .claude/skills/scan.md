# /scan — 全市场扫描

## 核心理念

不依赖实时行情 API（盘后不可用），通过**联网搜索 + AI 判断**发掘符合用户审美的标的。

用户审美：**周期底部 + 大盘股（市值>100亿）+ 有业绩支撑（ROE 为正、有盈利）**

## 执行步骤

### 1. 联网搜索当前市场状态

并行搜索以下方向，获取最新市场信息：

```
搜索 1: "A股 周期股 底部 低估值"
搜索 2: "A股 沪深300 低PE 大盘股"
搜索 3: "A股 行业轮动 周期底部板块"
搜索 4: "A股 破净股 高股息 大盘"
```

### 2. 提取候选标的

从搜索结果中提取：
- 哪些行业/板块处于周期底部？
- 具体哪些大盘股被提及为低估？
- PE、PB、股息率等关键数据
- 排除已在 watchlist 中的标的

### 3. 对 top 候选做量化验证

对最有价值的 5 个候选（不在 watchlist 中），用 `analyze.py` 跑量化数据：

```bash
cd /Users/zhaojunjie/pyProject/cycleLaboratory && python3 scripts/analyze.py --symbol <SYMBOL> 2>&1 | tail -1
```

注意：`analyze.py` 需要先在 watchlist.yaml 中添加该标的才能分析。如果候选不在 watchlist 中，先用 fetch.py 拉数据：

```bash
# 先临时加到 watchlist 或直接 fetch
python3 scripts/fetch.py --symbol <SYMBOL>
```

### 4. 输出格式

```
🔍 全市场扫描 — 2026-05-27

📰 当前市场状态：
- 一句话概括当前周期位置
- 哪些板块在底部，哪些在顶部

📊 候选标的：
1. 601XXX  XX银行    PE 4.5  市值 3500亿  行业: 银行
   - 周期逻辑：...
   - 风险点：...
   - 建议：值得加入 watchlist / 再观察 / 放弃

2. ...
```

### 5. 判断标准

对每个候选，回答三个问题：
- **是真周期底部还是价值陷阱？**（行业需求是否会恢复？）
- **业绩支撑是否扎实？**（ROE 稳定性、现金流、负债率）
- **是否值得加入 watchlist？**（给出明确 yes/no 和理由）

如果某个候选信息不足，诚实说"需要进一步研究该行业"。

## 风格

- 重点放在**不在 watchlist 里的新标的**
- 宁可少推，不推垃圾
- 区分"便宜但合理"和"便宜是因为要死了"
- 结合用户的周期价值投资风格做判断
